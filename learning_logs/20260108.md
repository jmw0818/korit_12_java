## ✅ 참조 자료형 캐스팅(Reference Type Casting)
- 객체 지향 프로그래밍(OOP)에서 중요한 요소 중 하나
- ch14_casting_animal   
-> Animal 클래스와 Dog, Cat 클래스와의 업캐스팅, 다운캐스팅 실습
- ch14_casting_centralcontrol  
-> CentralControl 클래스를 통한 device 통제 (with. power 인터페이스)  
-> AirConfitioner, Computer, LED, Mouse, Printer, Speaker device를 
  Power[ ] 자료형으로 필드를 선언해 deviceArray를 통해 전체적으로 관리
- poewrs 배열에 각 device별 객체를 집어넣어 업캐스팅(암시적)
- 다운캐스팅을 이용해 각 device의 고유메서드에 접근(명시적)

### 사전 지식
- Java 상에서의 자료형은 크게 두 가지로 구분됨.
  1. 기본 자료형(Primitive Type) : int, double, char, float, long
  2. 참조 자료형(Reference Type) : 객체를 참조하는 변수로, 클래스나 인터페이스로 생성
- 참조 자료형 캐스팅이란?  
-> 객체의 자료형을 변환하는 작업.
- int -> double / int -> long 으로 변경하는 것을 형변환(Conversion) 이라고 했음.

### 캐스팅 종류
1. Upcasting : 하위 클래스 타입의 객체를 상위 클래스 타입으로 변환
2. Downcasting : 상위 클래스 타입의 객체를 하위 클래스 타입으로 변환  
※ 상위클래스, 하위클래스를 어떻게 판단? -> 기본적으로 상속

### 미리 기억해두면 좋은 연산자(Operator) : instanceof (결과값은 기본적으로 boolean)
- 1 + 1 == 2 라고 했을 때 + 가 연산자에 해당하는 것처럼,
- 객체명 instanceof 클래스명 -> true / false가 튀어나옴. (조건문에서 쓰이겠구나)

### Upcasting
- 암시적으로 이루어짐 (알아서 바뀌기 때문에, 언제 알아서 바뀌는지 인지하고 있어야 함)

### Downcasting
- 명시적으로 이루어짐 (반드시 개발자가 적절한 장소와 시간대에 사용해야 함)

### 업캐스팅과 다운캐스팅의 활용
- 캐스팅은 다형성(Polymorphism)을 구현할 때 사용, 상위 클래스 타입으로 객체를 처리하면서도
특정 상황에서는 하위 클래스의 고유 기능을 사용 가능. -> 클래스 타입을 전환시키는게 중요한 이유는
우리가 만든 클래스의 객체를 기존 개발자들이 만든 클래스로 전환시킬 수 있다면 이미 만들어진 기능을
손쉽게 쓸 수 있기 때문.

### 장점
- 코드 유연성 : 업캐스팅을 통해 '다양한 객체를 하나의 상위 클래스 타입으로 관리' 가능
- 다형성 구현 : 메서드 오버라이딩과 함께 사용하면 코드의 재사용성 ↑
### 단점
- 다운캐스팅의 위험성 : 잘못 다운캐스팅을 하게 될수 있음.
    - 상위 클래스 A와 하위 클래스B, C가 있다고 가정할 때, B의 객체인 b를 A로 업캐스팅한
    이후에 C로 다운캐스팅을 시도하면 컴파일링 오류가 발생
- 복잡성 증가 : 코드 가독성이 떨어질 수 있음.
  - 바뀌는게 많아지기 때문인데 특히 업캐스팅의 경우 암시적으로 일어나기 때문에 어디서 이루어졌는지
  확인 못하고 있다가 갑자기 명시적으로 작성하는 다운 캐스팅이 튀어나와서 초심자들이 코드를 읽는게
  어려워지는 경우가 발생하기도 함.
- 이상의 단점을 보완하기 위해 나온 연산자가 instanceof 이다.

🛎️ 예시
```java
public static void main(String[] args) {
    System.out.println(objecA instanceof ClassA); // 결과값 : true / false
}
// objectA : 검사하려는 객체명
// ClassA : 객체가 검사될 클래스 / 인터페이스
```
return 값은 objectA가 ClassA의 인스턴스이거나 '하위' 클래스의 인스턴스인 경우 true,
아니면 false에 해당.
