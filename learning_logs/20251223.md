## ✅ Scanner
### 1. 기본 코드
```java
Scanner scanner = new Scanner(System.in);  // 객체 생성
scanner.nextLine();  // 메서드 호출
```
### 2. println vs print
ln  : line의 의미로 실행하고 마지막에 개행  
print로 작성하면 동일 라인에 그대로 연결됨

### 3. next (Int / Double / Boolean, ... etc) vs. nextLine()  
- nextLine()은 띄어쓰기를 허용. enter키 기준으로 데이터가 변수에 저장됨.  
-> 그래서 scanner.nextLine()으로 사용했을 떄 kim il이 모두 저장가능했음.  
- 나머지 (next)는 허용이 안돼서 kim 2를 입력했을 때, kim만 저장이 됨.  
- nextLine()이 엔터키 기준으로 입력되는 것은 어떤 부분에서 애매한 경우도 있다.
```java
System.out.print("올해는 몇 년도 인가요? >>> ");
int year = scanner.nextInt();
// 해결방법 : enter키를 받아주는 nextLine()을 하나 더 써줌.
scanner.nextLine();
System.out.print("당신의 이름은 무엇인가요? >>> ");
String name2 = scanner.nextLine();
System.out.println("연도 : " + year);
System.out.println("이름 : " + name2);
```

## ✅ Conversion
### 1. 업캐스팅(upcasting)  
- 더 큰 용량의 자료형으로 변환
```java
// 방법 # 1
char cast1 = 'A';
System.out.println("원형 : " + cast1);  // 결과값 : A
System.out.println("(int)변형 : " + (int)cast1);  // 결과값 : 65

char cast2 = 'a';
System.out.println("원형 : " + cast2);  // 결과값 : a
System.out.println("(int)변형 : " + (int)cast2);  // 결과값 : 97

// 방법 #2
char cast3 = 'b';
int casted3 = cast3;
System.out.println("원형 : " + cast3);  // 결과값 : b
System.out.println("변형(casted3) : " + casted3);  // 결과값 : 98
```

### 2. 다운캐스팅(downcasting)  
- 더 작은 용량의 자료형으로 변환
```java
int cast4 = 99;
int cast5 = 100;
// 방법 #1
System.out.println("원형 : " + cast4);  // 결과값 : 99
System.out.println("(char)변형 : " + (char)cast4);  // 결과값 : c

// 방법 #2
char casted5 = cast5;   // 오류 발생 -> (char) cast5;
System.out.println("원형 : " + cast5);  // 결과값 : 100
System.out.println("변형(casted5) : " + casted5);  // 결과값 : d
```
- 다운캐스팅을 진행할 때, 업캐스팅 #2 방식으로 처리하면 오류가 난다.  
  why? 업캐스팅의 경우 메모리 용량이 커지면 되어서 기존 데이터의 변형이 일어날 가능성 없이
  임시적(implicit)으로 변형이 가능하지만  
  다운캐스팅의 경우 테이터 용량이 작아지는 과정에서 데이터가 메모리 용량을 벗어나게 되는 경우가 발생.  
  그래서 미리 점검하기 위해 다운캐스팅 시에는 명시적(explicit)으로만 변경이 가능함.

## ✅ Condition (조건문)
### 1. if 문  
- if 다음에 있는 ( ) 내의 조건식이 true일 때 { } 내의 실행문이 실행됨.  
        false면 실행되지 않는다.
```java
if (조건식) {  // 조건식 :  true / false로 결론이 날 수 있는 식 또는 문장
    실행문;    // true일 때 실행
}
```
### 2. if - else 문  
- if 문 다음에 나오는 조건식이 true일 때는 if 내의 { } 실행문이 실행  
        / false일 때는 else에 딸린 { } 실행문이 실행
```java
if (조건식) {
    실행문 1;
} else {      // else의 경우 if가 false일 때만 실행되기 때문에
    실행문 2;  // if에서처럼 조건식이 따로 필요하지 않음.
}
``` 

### 3.if - else if 문  
- if - else 문과 달리 else if에는 별도의 조건식이 요구된다.  
        그리고 else if는 복수로 작성 가능하다.
```java
if (조건식 1) {
    실행문 1;
} else if (조건식 2) {
    실행문 2;
} else if (조건식 3) {
    실행문 3;
}
``` 
-> 조건식에 다 해당하지 않는 경우 원하는 결과가 나오지 않을 수 있다.

### 4. if - else if - else 문
```java
if (조건식 1) {
    실행문 1;
} else if (조건식 2) {
    실행문 2;
} else if (조건식 3) {
    실행문 3;
} else {
    실행문 4;
}
```
